# Mac 输入法锁定工具实现方案

## 概述

该工具采用"监听-纠正"模式，在 macOS 系统中实现输入法锁定功能。由于 macOS 没有提供真正的输入法锁定 API，该方案通过监听系统输入法变化通知，并在检测到非预期切换时自动纠正，达到等效的锁定效果。

---

## 核心原理

### 监听-纠正模式

1. **监听阶段**：持续监听系统级输入法变化通知
2. **检测阶段**：比较当前输入法与用户锁定的目标输入法
3. **纠正阶段**：发现不一致时，自动切换回锁定的输入法

---

## 技术实现

### 系统依赖

- **macOS Text Input Sources (TIS) API**：Apple 官方提供的输入法管理接口
- **DistributedNotificationCenter**：系统级通知中心，用于接收输入法变化事件

### 核心工作机制

| 组件 | 功能 |
|------|------|
| 输入法列表获取 | 枚举系统中所有可用的输入法 |
| 当前输入法检测 | 实时获取当前激活的输入法 |
| 输入法强制切换 | 程序化切换到指定输入法 |
| 变化事件监听 | 响应系统输入法变化通知 |

## 关键设计要点

### 1. 50 毫秒延迟

- **目的**：确保系统完成输入法切换操作后再执行纠正
- **效果**：避免与系统切换过程产生冲突，保证纠正操作可靠执行

### 2. ID 比较

- 使用输入法的唯一标识符（ID）而非名称进行比较
- 避免因同名输入法导致的误判

### 3. 异步处理

- 使用异步队列处理纠正操作
- 避免阻塞主线程，保证界面响应流畅

## 方案优势

| 优势 | 说明 |
|------|------|
| 无需系统权限 | 使用公开 API，无需特殊系统权限 |
| 响应迅速 | 50 毫秒延迟对用户几乎无感知 |
| 兼容性好 | 基于系统标准接口，兼容性稳定 |
| 实现简洁 | 代码量少，维护成本低 |

---

## 方案局限

| 局限 | 影响 |
|------|------|
| 非真正阻止 | 用户仍可触发切换，只是立即被纠正 |
| 短暂闪烁 | 在某些情况下可能短暂显示切换过程 |
| 系统依赖 | 依赖 macOS TIS API，未来系统更新可能需要适配 |

---

## 总结

该方案通过巧妙的"监听-纠正"模式，在没有原生输入法锁定 API 的 macOS 上实现了等效功能。其核心思想是：既然无法阻止用户切换输入法，就在检测到切换后立即将其纠正回锁定状态。从用户体验角度，50 毫秒的响应延迟使得这种纠正几乎不可感知，达到了预期的锁定效果。
